12. Warm Up Packet Tracer and IOS Basic

IOS 是思科網路的系統名稱

Bring Up a Device

Router Bootup Process

    1. Perform POST
    • Power-on Self-test(POST) - 通電自檢
    • 對於硬件做基本檢測
    1. Load Bootstrap
    • 當當於電腦的BIOS
    1. Locate IOS
    • 啟動系統二種方法
      • 啟動一個文件，存在於flash上，相當於本地硬盤一樣
      • 存取TFTP Server 上的文件進行啟動
        • 跟無盤工作站概念很相似
    1. Load IOS
    • 加載系統
    • 加載完過後，要加載配置文件
    1. Locate the Configuration File
    • NVRAM
    • TFTP Server
    • Console
    1. Execute the Configuration File or enter “Setup” mode

NVRAM 斷電之後，儲存在裡面的資料不會流失

Different Modes

USER MODE

show users
show version
無法進行配置

PRIVILEGED MODE

enable
也無法進行配置
?
接口地址、接口路由，在這邊也無法配置

CONFIGURE MODE

指令
configure terminal
exit
或者打
config
t

可以對系統配置進行修改
要配置接口就必須在 interface 模式以此類推…

Viewing,Saving, and Erasing Configurations

Running configuration
正在敲打配置命令時，是存在於RAM裡面的

要經過
copy running-config startup-config
才會寫到NVRAM

系統reboot時都是找Startup configuration

系統上敲打wr 跟copy running-config 效果是一樣的

Configuration Register

IOS 要去找starup-config 是由configuration register 控制的
改完之後不會馬上生效
在這個模式要加上do 指令才能show

Boot System

系統要從哪裡啟動

  • 本地硬碟
  • TFTP

可以先備份舊系統到別台服務器，以便還原
請操作練習

ROM Monitor(ROMMON)

作業系統丟失時，比方說斷電導致IOS死掉，可以用ROM Monitor 進行recover

圖中所列出的指令，記得大小寫是有差的。

  • copy

  • show file

  • delete

  • erase

  • cd/pwd

  • mkdir/rmdir

  • ctrl + a

  • ctrl + e

  • ctrl + u

軟件三原則的個人理解
在軟件的設計當中前人已經總結了許多的設計原則和設計模式。例如SOLID，GRASP設計原則，這些原則都是基於面向對象設計總結而來的。而GOF23是基於許多常見的場景總結出了一套設計模式，在我們遇到類似的場景，都可以套用設計模式。而今天所講到的軟件三原則是適用於在軟件設計的各個層面的。它不僅適用於面向對象的設計，也適用於面向過程的程序設計；不僅適用於類的設計，也適用於模塊、子系統的設計。就連在項目架構運維部署中也適用於這一套簡單的法則。
