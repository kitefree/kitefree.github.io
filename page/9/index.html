發表於 2021-05-30 更新於 2023-12-31 分類於 負載平衡
400 1 分鐘

1. 什麼是負載平衡?

  • 頻寬管理

    • LAN Load Balance
    • WAN Load Balance
      WAN Load Balance（WLB），一般也稱為Multi-Homing設備。當企業對外線路有兩條以上的時候，WLB就可以擔負起網際網路（Internet）與內部網路（Intranet）之間多條連外線路流量平衡的工作。 LAN Load Balance針對區域網路，設置在封包類別較複雜的節點，有助於調整、控制封包的進出順序。

  • 應用管理

    • Server Load Balance
      Server Load Balance（SLB）是最早發展的一塊，其功能在於將企業內多部伺服器的負載量作平衡，將過大流量轉向相同功能的其他伺服器，讓每一部伺服器工作量維持平均，保持服務不因流量過大而變慢或中斷。SLB主要應用在入口端的網頁分流，透過SLB轉址到各個伺服器，平均分配網路流量。

12. Warm Up Packet Tracer and IOS Basic

IOS 是思科網路的系統名稱

Bring Up a Device

Router Bootup Process

    1. Perform POST
    • Power-on Self-test(POST) - 通電自檢
    • 對於硬件做基本檢測
    1. Load Bootstrap
    • 當當於電腦的BIOS
    1. Locate IOS
    • 啟動系統二種方法
      • 啟動一個文件，存在於flash上，相當於本地硬盤一樣
      • 存取TFTP Server 上的文件進行啟動
        • 跟無盤工作站概念很相似
    1. Load IOS
    • 加載系統
    • 加載完過後，要加載配置文件
    1. Locate the Configuration File
    • NVRAM
    • TFTP Server
    • Console
    1. Execute the Configuration File or enter “Setup” mode

NVRAM 斷電之後，儲存在裡面的資料不會流失

Different Modes

USER MODE

show users
show version
無法進行配置

PRIVILEGED MODE

enable
也無法進行配置
?
接口地址、接口路由，在這邊也無法配置

CONFIGURE MODE

指令
configure terminal
exit
或者打
config
t

可以對系統配置進行修改
要配置接口就必須在 interface 模式以此類推…

Viewing,Saving, and Erasing Configurations

Running configuration
正在敲打配置命令時，是存在於RAM裡面的

要經過
copy running-config startup-config
才會寫到NVRAM

系統reboot時都是找Startup configuration

系統上敲打wr 跟copy running-config 效果是一樣的

Configuration Register

IOS 要去找starup-config 是由configuration register 控制的
改完之後不會馬上生效
在這個模式要加上do 指令才能show

Boot System

  • TFTP

可以先備份舊系統到別台服務器，以便還原

ROM Monitor(ROMMON)

作業系統丟失時，比方說斷電導致IOS死掉，可以用ROM Monitor 進行recover
