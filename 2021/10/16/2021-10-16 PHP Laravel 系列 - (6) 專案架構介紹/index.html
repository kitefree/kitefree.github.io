PHP Laravel 系列 - (6) 專案架構介紹

發表於 2021-10-16 更新於 2022-08-24 分類於 PHP
1.1k 2 分鐘

完整專案架構截圖

app structure

在windows 要輸出這樣的結構可以使用cmd指令產出

tree > dir.txt /a /f

app

程式最常修改的地方

Console

可以寫一些指令相關的檔案，前期暫時不會使用到它

Exceptions

做一些除錯相關的檔案，前期暫時不會使用到它

http

這是一個很重要的folder

controllers

使用者進入程式的進入點設定

Middleware

比較進階的模塊

Models

擺放一些資料模塊

Providers

有些邏輯可以包成一些服務，可以放在這兒。

或者是套件的一些使用方式都會在這邊

bootstrap

是比較進階的，跟程式運作有關，很少會碰到它

config

可以去設定一些參數，比方說要連到哪個資料庫、email要發到哪個server ip

database

跟資料庫操作有關的，或者是要做一些假資料，或者是啟動server時預設產生的資料等等

public

放的是靜態檔案 比方說圖片或pdf等等

resource structure

css

可以放一些前端必備的檔案

js

可以放一些前端必備的檔案

views

view有關的程式碼

routes

使用者透過網址拜訪網站，透過路由決定要執行哪一段程式，下一章節會介紹

storage

使用者上傳檔案，或者是你要產生暫時檔案，都在這邊哦

tests

進行程式測試使用的

vendor

這個專案用到的套件

# PHP # Laravel
五個必須知道的命名規則
