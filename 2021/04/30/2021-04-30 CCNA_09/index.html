8. Subnetting 1

Subnetting 劃分子網，大的網路地址劃分地小的網路地址範圍

為什麼要劃分?

  • More Secure
    • 網路安全
      HR + IT + Sale 如果都在同一個網域裡，就可以直接訪問IT，顯然不安全…
  • Easier to Manager
    • 好管理
      • 比如針對IT部門做決策政策管理，就可以直接找同網段的IP即可
  • Save IP Address
    • 節省地址空間
      以下範例說明了浪費，Class C 2^8 - 2 個地址可用
      • 192.168.10.0 HR
      • 192.168.20.0 IT
      • 192.168.30.0 Sale

Why you need subnetting

把一個網段劃分成若干網段

How to create subnets

Classful 沒進行子網劃分稱為有類
Classless 進擁子網劃分稱為無類
複習一下
Class A 8 + 24
Class B 16 + 16
Class C 24 + 8

Subnet Mask

給你這3個ip 是無法判斷是否為同網段

  • 192.168.1.1
  • 192.168.1.129
  • 192.168.1.254

只有IP Address + Subnet Mask 才能判斷是否為同一個網段

Classless Inter-Domain Routing (CIDR)

Subnet Mask
不能迅速告訴我借了多少bit給network，所以可以用CIDR
CIDR
又稱為Slash Notation
Prefix
/8 代表有8個網路位
以/26為例

Subnet Mask with CIDR

思科設備大部分是以MASK 進行設定，其它廠商有時候會以CIDR進行設定

  • IP/32
    • 代表是一個主機
  • IP/30
    • 2^-2 = 2
    • 最小網段
    • 如何確認另一個IP是哪一個呢?
      • Tips
      • the lower IP is always odd,and the higher is always even
  • IP/31
    • 是理論上最小的網段
    • 2個HOST彼此之間溝通，不需要再有頭跟尾的IP需求(網路、廣播)
    • 能否設定IP/31 要看硬體設備是否有支持這樣的作法

Subnet Zero

思科設備在版本12.x 以前 subnet zero 是不允許使用的
